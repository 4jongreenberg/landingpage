The Stack

We have taken the best of the web and put it into one very cool package. Below you'll find an quick explanation of everything that makes up the Webplate stack and how it executes.

Lead by example

Before we get into the nity gritty its important to note that an example HTML file is found in the root download folder called webplate-basic.html. From here you can start creating any website or webapp using webplate. For more advanced layouts check the other webplate- something files. They range from app layouts to blogs and have all the required extra CSS and Javascript already added in.

Load the neccessary files

To make any page use webplate all you need to do is drop a copy of the webplate directory you got in the download into the root or any subdirectory you have in your site. Then go ahead and include two files that will execute the entire webplate stack. Firstly you will need the webplate.css followed by the webplate.js files. The CSS is loaded first so that the page renders properly before the javascript finishes doing its thing. If you include the CSS afterwards then the page will render out as pure text before everything aligns in place with colours and layouts. See an example of a typical include below (This example assumes that webplate is in the site root):

<head>
    <link href="webplate/css/webplate.css" type="text/css" rel="stylesheet" />
    <script src="webplate/js/webplate.js" type="text/javascript"></script>
</head>

First the CSS

The CSS is a compressed and minified version of a LESS file found under webplate/less/ and is an combination of many files importing into one via webplate-imports.less. If you wish to include more of your own files into this core set, you can by creating a new LESS file and adding the import link (Note that it won't be included in any future updates and would need to be relinked every time). You can instead use an extras include but that will be explained a bit later.

I recommend you learn about LESS via the website lesscss.org. While webplate uses LESS it's important to note that you do NOT NEED any experience in it and that if you want you can style directly in CSS alone without any concerns.

So in this CSS file is a combination of things. Firstly we include the global webplate.less file that has a series of helper mixins like rounded corners or drop shadows. Include this file in any custom LESS files you create to access these useful helper mixins and to gain access to the @page_width variable that is crucial to webplates responsive media query checks.

Next webplate imports Normalize which is a series of CSS resets and global styles to make cross-browser rendering more accurate. It kind of levels the styling playing field for you.

Once the resets have taken place, webplate loads up Typeplate that essentially sets up a series of typographical parameters that makes your text look neat and organized right out of the webplate box.

This is followed by webplate global, a small LESS file of styles and useful classes that help manage some of the more universal elements.

Right afterwards the webplate layout LESS file is loaded and is the heart and soul of all the responsive goodness that happens on a webplate site or app. This is achieved through the usage of media queries and class names and is explained in more detail in the layout section.

Next up is the styling and resets of forms down through the webplate forms LESS file. The cool thing about this inclusion is the global nature of all your form elements, which includes all browsers and all mobile devies too.

Last but not least, we import the buttons LESS file that allows you to create awesome looking buttons quickly and easily. More information on its usage can be found in the buttons section.

Layout
