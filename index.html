The Stack

Webplate takes the best of the web and puts it into one very cool package. Below you'll find a quick explanation of everything that makes up the Webplate stack and how it executes.

Lead by example

Before we get into the nitty gritty it's important to note that an example HTML file is found in the root download folder called layout-basic.html. From here you can start creating any website or web app using webplate.

Load the necessary file (Yup, just one)

To make any page use webplate all you need to do is drop a copy of the webplate directory you got in the download into the desired directory of your site. Then go ahead and include only one file (used to be two) that will execute the entire webplate stack, named stack.js. See an example of a typical include below (This example assumes that webplate is in the site root):

<head>
	<script id="webplate-stack" src="webplate/stack.js" type="text/javascript"></script>
</head>

First the CSS

The CSS is a compressed and minified version of a LESS file found under webplate/core/less/ and is an combination of many files importing into one via webplate-imports.less. I recommend you learn about LESS via the website lesscss.org. While webplate uses LESS it's important to note that you do NOT NEED any experience in it and that if you want you can style directly in CSS alone without any concerns.

So in this CSS file is a combination of things. Firstly we include the global webplate.less file that has a series of helper mixins like rounded corners or drop shadows. Include this file in any custom LESS files you create to access these useful helper mixins and to gain access to the @page_width variable that is crucial to webplates responsive media query checks. Next webplate imports Normalize which is a series of CSS resets and global styles to make cross-browser rendering more accurate. It kind of levels the styling playing field for you. Once the resets have taken place, webplate loads up Typeplate that essentially sets up a series of typographical parameters that makes your text look neat and organized right out of the webplate box.

After that the webplate layout LESS file is loaded and is the heart and soul of all the responsive goodness that happens on a webplate site or app, followed by the webplate forms LESS file. The cool thing about this inclusion is the global nature of all your form elements, which includes all browsers and all mobile devices too. Last but not least, we import the buttons LESS file that allows you to create awesome looking buttons quickly and easily. More information on its usage can be found in the buttons section.

Now the javascript

Once the CSS is loaded, the webplate javascript loads and executes off quite a bit. This file is required and includes a couple of compulsory features. Firstly, webplate runs off the jQuery javascript framework and cannot function without it. It also loads Modernizr and some webplate tools and does so using yepnope.js. It is in this file that the navigation and page wrappers are set, the touch device check is determined (jquery.finger.js is loaded if the device has touch capabilities) and the extras are loaded.

What are extras?

Extras are a way to add in your own css or javascript and never have to load them manually in the header. Why is this useful? Well one it makes it easy to manage these files and keeps your code very neat and two it is just the beginning of a dependency manager that is being developed for webplate. In the current webplate for example, you can add in a CSS or a LESS file and if webplate detects the LESS, then it will load the required javascript to compile your code client side. The files are stored in the root webplate/extras directory. To use the extras option, add the relevant data attributes into the page's body tag. See the example below:

<body data-css-extras="welcome.css" data-js-extras="welcome.js">

Note: Use a comma separator to add in multiple files of any one type.

Layout

Using a fluid responsive layout and integrating a horizontal 12 column system, you can best display your content in three various views, large, adjusting and small. Vertically everything is displayed in rows that together with columns create blocks that respond to different dimensions. Why not read about the different options below. Please note that the padding between spans is only for this example and not a default setting.

View types

At its base level, webplate has 3 view types (although you can add more yourself). These are Large View, Adjusting View and Small View. Large and adjusting views are similar, with the only difference being the @page_width variable found in the webplate/core/less/webplate.less file. By default this variable is set to 1100px and is there to stop your content from displaying extremely wide on big high resolution screens. In essence this is the maximum content width that will be allowed. If the screen scales down to a width less than 1100px, the content will become fluid and snap to 100% width, making all content within begin to adjust accordingly.

Of course this variable is completely editable and can be set to whatever you want it to be. In the case of web apps you can set it to 100% to fill your entire screen real estate which will essentially remove the minor difference between the two. You will also see that as the screen scales that the webplate javascript adds a class to the html tag. In the case of large and adjusting view, the class is web-large-view while in the case of small view the class is web-small-view. This is extremely useful when styling your site or app.

Once the page scales down to less than and including 700px, webplate snaps into small view which is really the mobile device domain. All content will now be displayed vertically unless you state otherwise (read below for more on this).

Please note that webplate does NOT follow a mobile first structure philosophy because this methodology is somewhat redundant in technical terms. Webplate provides a framework to display information in mobile, fluid and desktop based outputs and the mobile first philosophy should be a critical part of your UX design process rather than a working constraint of the framework itself.

Navigation

To create a navigation that slots into the page slider, class the containing element with the class navigation. Webplate will then clone it into the page slider that is show only when any element with the class navigation-trigger is clicked. View this page's navigation to see this effect. Below is a basic example:

<div class="navigation-trigger icon-reorder"></div>
<nav class="navigation">
	<ul class="row">
		<li class="span-2">
			<a href="#">Navigation Link 1</a>
		</li>
		<li class="span-2">
			<a href="#">Navigation Link 2</a>
		</li>
	</ul>
</nav>

How to center content (.limit)

In order to center content you need to use the limit class that will contain any child elements to the @page_width (default 1100px) and automatically center it. You can do so to all your content by using a wrapper div or only certain parts of your site like this one. In the case of webplate each row has a background colour and is a 100% width with the limit content wrapped in the limit tag. See examples below:

<body>
	<div class="limit">Anything inside here will be centered.</div>
</body>

In the case of this site:

<body>
	<div class="row">
		<div class="limit">
			This content will be centered while the outer row will be at a 100% width.
		</div>
	</div>
</body>

Offset (.offset-*)

Sometimes you might not always want to use the full width of the columns available for whatever reason. In these cases offsetting becomes necessary and moves the classed element to the right of the element preceding it based on the amount of columns that you specify. See an example below:

<div class="row">
	<div class="span-4">Span 4</div>
	<div class="span-4 offset-2">Span 4, Offset 2</div>
</div>
Span 4
Span 4, Offset 2

Offset Right (.offset-right-*)

Alternatively you could push elements off to the right from the classed element by using offset right.

<div class="row">
	<div class="span-4 offset-right-1">Span 4, Offset Right 1</div>
	<div class="span-6">Span 6</div>
</div>
Span 4, Offset Right 1
Span 6

The possibilities are endless

You can play around with 3 column and 4 column layouts and really the choices is quite varied. The columns are percentages of the row and when in a fixed width layout are statically constrained. Once you scale down into a fluid adjusting view, the span classed elements will begin to adjust and move according to the space available. Once you get to small view though everything will stack vertically meaning that all spans will become 100% width and make all content easier to read on mobile devices.

But I want to span in small view?

No problem! By adding in the prefix small- to either the span or offset classes you keep the same functionality in small view. See the example below (You will need to resize the view to see it take effect):

<div class="row">
	<div class="span-4 small-span-4">Span 4, Small Span 4</div>
	<div class="span-7 small-span-7 offset-1 small-offset-1">Span 7, Small Span 7, Offset 1, Small Offset 1</div>
</div>
Span 4, Small Span 4
Span 7, Small Span 7, Offset 1, Small Offset 1

Small hide (.small-hide)

There are a few classes which are helpful at this stage to know, namely small-hide. This will hide the classed element in small view while displaying it in large view.

Small show (.small-show)

Inversely small-show will display the classed element in small view whilst hiding it in large view.

Forms

The challenge of globally styled form elements is a big one especially when you start to consider mobile devices and tablets including the normal desktop browser. For this reason webplate has some forms overrides and styling to make them all look the same and work universally.

Choosing the colour for you

There are currently 9 colour options to choose from when applying styles to your site or app. These are red, green, blue, aero, grey, orange, yellow, pink and purple and are derived from the iCheck library used by webplate. Webplate has integrated the colour schemes into the other forms elements like input, textareas and dropdowns. By default the colour is set to blue, but is easily changed by adding a data attribute to the body tag. See an example below.

<body data-forms-colour="orange">

Making it all work

Webplate will do all the work for you automatically, however if you want labels for your checkboxes or radio buttons then a wrapper div needs to be added with the class inline. See the example below of what you would need to do:

<div class="inline">
	<input type="checkbox">
	<label>Checkbox 1</label>
</div>

IcoMoon Icon Fonts

Webplate uses IcoMoon for its icon fonts because not only is it free, but also rather extensive with over 3,800+ icons to choose from. Gone are the days of tons of images to display icons and graphics and the best part is, because they are resizable and vector based, they are great for responsive sites.

Use an icon font

Most of the time icon fonts are used by classing a span tag but ultimately can be added to any tag you wish. They are subject to any styling that can be applied to text like colour, size and even something more advanced like text-shadow. See an example of its usage below:

<div><span class="icon-location"></span> Some Text</div>
<div><span class="icon-stack"></span> Some More Text<div>

Some Text
Some More Text

Updating the IcoMoon library

There is a small base set of icons included but ultimately more will need to be added to any web project. To do so sign up to IcoMoon and start by loading the current session file into your profile. This file is found under the webplate/core/icomoon directory with the filename icomoon-session.json. From this point start adding your own icons in and select the fonts link below before downloading. From there you will need to copy the contents of that download package into the webplate/core/icomoon directory. Click the View Webplate Icons button below to see your added fonts.
